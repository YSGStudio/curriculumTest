# 2022 개정 교육과정 성취기준 웹 서비스 기획서

> **스택:** Next.js 14 (App Router) · React · Tailwind CSS · Supabase  
> **목적:** 3~4학년군 성취기준 탐색 + 수업 재구성 워크숍 기능을 웹 서비스로 구현  
> **데이터 전략:**  
> - 성취기준 CSV → 프로젝트 내장 (빌드 타임 파싱, DB 불필요)  
> - 수업 재구성 계획서 → Supabase DB 저장 (로그인 필요)

---

## 1. 프로젝트 개요

### 배경
- 2022 개정 교육과정 성취기준(231개)을 교사가 빠르게 검색·탐색할 수 있는 도구 필요
- 성취기준 데이터는 변하지 않으므로 CSV를 프로젝트에 내장해서 사용 (DB 불필요)
- 수업 재구성 계획서는 Supabase에 저장하여 어느 기기에서나 불러올 수 있도록 구현

### 핵심 기능
| 기능 | 방식 | 로그인 필요 여부 |
|------|------|----------------|
| 성취기준 탐색 | CSV 내장 (빌드 타임 파싱) | ❌ 불필요 |
| 과목 필터 | 클라이언트 상태 | ❌ 불필요 |
| 키워드 검색 | 클라이언트 실시간 필터 | ❌ 불필요 |
| 아코디언 UI | React 상태 | ❌ 불필요 |
| 성취수준 A/B/C 보기 | 정적 렌더링 | ❌ 불필요 |
| 수업 재구성 작성 | 클라이언트 상태 | ❌ 불필요 |
| 수업 재구성 저장·불러오기 | Supabase DB | ✅ 필요 |

### 3-2. CSV 컬럼 구조

| 컬럼 | 내용 | 예시 |
|------|------|------|

| 핵심 아이디어 | 영역 핵심 아이디어 전문 | ⋅듣기⋅말하기는 ... |

### 3-3. 빌드 타임 파싱 (`lib/loadCriteria.ts`)

```ts
// 서버 컴포넌트에서만 실행 — 클라이언트 번들에 CSV 원본 미포함
import fs from 'fs'
import path from 'path'
import { parse } from 'csv-parse/sync'
import type { Subject } from '@/types'

export function loadCriteria(): Subject[] {
  const filePath = path.join(process.cwd(), 'data', 'criteria_3_4.csv')
  const fileContent = fs.readFileSync(filePath, 'utf-8')

  const rows = parse(fileContent, {
    columns: true,
    skip_empty_lines: true,
    bom: true,
  })

  const subjectMap = new Map<string, any>()

  for (const row of rows) {
    const code = row['성취기준 코드'].replace(/[\[\]]/g, '').trim()
    if (!code.match(/^4[가-힣]\d{2}-\d{2}$/)) continue

    const subjectName = row['교과'].trim()
    const domainName  = row['영역'].trim()

    if (!subjectMap.has(subjectName)) {
      subjectMap.set(subjectName, { name: subjectName, domains: new Map() })
    }
    const subject = subjectMap.get(subjectName)

    if (!subject.domains.has(domainName)) {
      subject.domains.set(domainName, {
        name: domainName,
        coreIdea: row['핵심\n아이디어']?.trim() ?? '',
        criteria: [],
      })
    }

    subject.domains.get(domainName).criteria.push({
      code,
      text:        row['성취기준'].trim(),
      levelA:      row['성취수준(A)'].trim(),
      levelB:      row['성취수준(B)'].trim(),
      levelC:      row['성취수준(C)'].trim(),
      explanation: row['해설'].trim(),
    })
  }

  return Array.from(subjectMap.values()).map(s => ({
    ...s,
    domains: Array.from(s.domains.values()),
  }))
}
```

## 4. Supabase DB 설계 (수업 재구성만)

### 4-1. 테이블

#### `lesson_plans` (수업 재구성 계획서)

```sql
create table lesson_plans (
  id                  uuid default gen_random_uuid() primary key,
  user_id             uuid references auth.users(id) on delete cascade,
  title               text not null,
  criteria_codes      text[],   -- 선택된 성취기준 코드 배열 e.g. '{4국01-01, 4수03-02}'
  integrated_standard text,     -- 통합 성취기준
  key_activities      text,     -- 핵심 활동
  schedule            text,     -- 차시 계획
  assessment          text,     -- 평가 방법
  memo                text,
  created_at          timestamptz default now(),
  updated_at          timestamptz default now()
);
```

### 4-2. Row Level Security (RLS)

```sql
-- 본인 데이터만 접근 가능
alter table lesson_plans enable row level security;

create policy "본인 계획서만 조회"
  on lesson_plans for select
  using (auth.uid() = user_id);

create policy "본인 계획서만 삽입"
  on lesson_plans for insert
  with check (auth.uid() = user_id);

create policy "본인 계획서만 수정"
  on lesson_plans for update
  using (auth.uid() = user_id);

create policy "본인 계획서만 삭제"
  on lesson_plans for delete
  using (auth.uid() = user_id);
```

### 4-3. updated_at 자동 갱신 트리거

```sql
create or replace function update_updated_at()
returns trigger as $$
begin
  new.updated_at = now();
  return new;
end;
$$ language plpgsql;

create trigger lesson_plans_updated_at
  before update on lesson_plans
  for each row execute function update_updated_at();
```

## 8. 수업 재구성 저장/불러오기 (`hooks/useWorkshop.ts`)

```ts
'use client'
import { useState } from 'react'
import { createClient } from '@/lib/supabase/client'
import type { LessonPlan, Criteria } from '@/types'

export function useWorkshop() {
  const supabase = createClient()
  const [selectedCriteria, setSelectedCriteria] = useState<Criteria[]>([])
  const [savedPlans, setSavedPlans]             = useState<LessonPlan[]>([])
  const [isPanelOpen, setIsPanelOpen]           = useState(false)

  // 저장된 계획서 불러오기
  const fetchPlans = async () => {
    const { data } = await supabase
      .from('lesson_plans')
      .select('*')
      .order('created_at', { ascending: false })
    if (data) setSavedPlans(data)
  }

  // 새 계획서 저장
  const savePlan = async (plan: Omit<LessonPlan, 'id' | 'user_id' | 'created_at' | 'updated_at'>) => {
    const { data: { user } } = await supabase.auth.getUser()
    if (!user) return { error: '로그인이 필요합니다.' }

    const { error } = await supabase.from('lesson_plans').insert({
      ...plan,
      user_id: user.id,
    })
    if (!error) await fetchPlans()
    return { error: error?.message }
  }

  // 계획서 삭제
  const deletePlan = async (id: string) => {
    await supabase.from('lesson_plans').delete().eq('id', id)
    setSavedPlans(prev => prev.filter(p => p.id !== id))
  }

  // 성취기준 선택 토글
  const toggleCriteria = (c: Criteria) => {
    setSelectedCriteria(prev =>
      prev.find(x => x.code === c.code)
        ? prev.filter(x => x.code !== c.code)
        : [...prev, c]
    )
  }

  return {
    selectedCriteria, toggleCriteria,
    savedPlans, fetchPlans, savePlan, deletePlan,
    isPanelOpen, setIsPanelOpen,
  }
}
```
